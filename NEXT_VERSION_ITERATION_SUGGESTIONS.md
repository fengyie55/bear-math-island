# 小熊数学岛下一版本（V6）迭代建议

## 1. 当前代码现状（快速评估）

- **分支同步现状**：当前本地分支 `work` 没有配置上游分支，无法直接 `git pull`，需要先绑定 `remote/branch` 后再同步。
- **技术结构现状**：应用入口非常轻，核心业务基本集中在 `GameUI.vue` 单文件组件；该文件已达到 2700+ 行，后续迭代会明显受维护成本影响。
- **文档与代码存在轻微偏差**：README 中描述了 `stores/`、`utils/` 目录，但仓库当前并未提供这些目录（逻辑主要仍在组件内）。

---

## 2. V6 迭代目标（建议）

建议将下一版本目标定义为：

1. **稳定性升级**：降低单文件复杂度，降低回归风险。
2. **学习效果升级**：把“推荐练习”从规则推荐升级为“薄层自适应推荐”。
3. **家长价值升级**：把“看报表”升级为“可执行陪练计划”。
4. **可运营升级**：增加内容配置化能力，缩短活动/题库更新周期。

---

## 3. 版本功能建议（按优先级）

## P0（必须做）

### 3.1 架构重构：拆分 GameUI 单文件

**现状问题**
- `GameUI.vue` 同时承担：题目生成、状态管理、音频、存储、成就、家长端、样式等职责。

**建议方案**
- 组件分层：
  - `screens/`：child-select / main / game / practice / parent
  - `components/game/`：QuestionCard、ChoiceGrid、FeedbackBar、StreakMeter
- 逻辑抽离：
  - `composables/useQuestionEngine.ts`
  - `composables/useAudioFeedback.ts`
  - `composables/useProgress.ts`
  - `composables/usePersistence.ts`
- 状态管理落地 Pinia：
  - `stores/profile.ts`
  - `stores/session.ts`
  - `stores/content.ts`

**验收标准**
- 单文件不超过 500 行。
- 关键逻辑（题目生成、推荐、存储迁移）具备单测。

### 3.2 自适应练习（Adaptive Lite）

**目标**
- 在“推荐”基础上，增加“知识点维度 + 错题重练 + 难度动态调整”。

**建议实现**
- 为每道题打标签：`domain`（加减乘除/比较/数数）、`difficulty`、`strategy`。
- 每次作答记录：正确性、反应时间、是否提示后作答。
- 推荐策略：
  - 最近 20 题正确率 < 70% 的知识点优先。
  - 连续正确 5 题升难度；连续错误 2 题降难度并触发讲解题。

**用户可见收益**
- 减少“会的题重复做”，提升单位学习时长的有效性。

### 3.3 测试与质量门禁

**建议最小闭环**
- 单测：题目生成边界（选项去重、范围合法、答案可达）。
- 组件测试：答题流（开始、作答、反馈、结算）主路径。
- CI：`npm run build` + `npm run test` 作为合并门禁。

---

## P1（强烈建议）

### 3.4 家长端升级为“行动建议”

- 增加“本周 3 个可执行任务”：
  - 例如“本周每天 8 题减法”“睡前口算 5 分钟”。
- 增加“问题诊断卡片”：
  - 如“借位减法错误率 42%（上升）”，附干预建议。
- 增加“陪练脚本”：
  - 一键生成 3 分钟口头游戏（家长可直接照读）。

### 3.5 内容配置化（降低运营成本）

- 把场景库、题干模板、奖励文案迁移到 `content/*.json`。
- 预留“节日主题包”机制（春节、开学季、六一）。
- 支持配置热更新（先本地配置化，后续可接远端配置）。

### 3.6 数据埋点与版本对比

- 关键漏斗：进入主页 → 开始练习 → 完成 1 局 → 次日回访。
- 学习指标：正确率、平均用时、连续学习天数、知识点覆盖率。
- 版本对比：V5 vs V6 的学习效率提升（每 10 分钟正确题数）。

---

## P2（可排后）

### 3.7 多角色与课堂场景

- 支持“家庭多孩对比视图”（仅家长可见）。
- 新增“老师模式（班级版）”预研：导入学生、分配练习包、导出周报。

### 3.8 AI 语音互动（实验）

- 当前语音播放基础上，扩展“鼓励语 + 解释语”的智能生成。
- 注意离线策略和儿童场景安全词库。

---

## 4. 建议排期（6 周）

- **第 1-2 周**：架构拆分 + 测试基线 + 埋点补齐。
- **第 3-4 周**：Adaptive Lite + 家长行动建议。
- **第 5 周**：内容配置化 + 主题活动包。
- **第 6 周**：灰度验证 + 指标复盘 + 文档收口。

---

## 5. 风险与规避

- **重构风险**：拆分时行为变化。
  - 规避：先补测试，再重构；采用“新旧并行开关”逐步切换。
- **推荐失效风险**：样本不足导致推荐抖动。
  - 规避：前 3 天走保守策略（规则优先），之后启用自适应。
- **内容质量风险**：题干模板多样性不足。
  - 规避：为每个知识点至少准备 20 条模板并做可读性抽检。

---

## 6. 本次建议的落地顺序（最小可行）

1. **先做工程治理（拆分 + 测试 + CI）**，保证后续迭代可持续。
2. **再做学习效果（Adaptive Lite）**，直接提升核心指标。
3. **最后做家长行动化与内容配置化**，扩大长期留存和运营效率。
